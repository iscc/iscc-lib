# Spec: C# / .NET Bindings — P/Invoke via csbindgen

A .NET library `Iscc.Lib` on NuGet providing idiomatic C# access to all ISCC functions via P/Invoke
over the existing C FFI. Built on top of `iscc-ffi` — the same shared library and `iscc.h` header
that powers C/C++ integration.

## Architecture

**Three-layer design:**

1. **C FFI layer** (`crates/iscc-ffi/`): Existing Rust `extern "C"` functions with `iscc.h` header.
    Already production-ready with 85 Rust + 65 C tests.
2. **P/Invoke layer** (`NativeMethods.g.cs`): Auto-generated by `csbindgen` from `iscc.h`. Raw
    `[DllImport]` declarations mapping C functions to .NET.
3. **Idiomatic wrapper** (`IsccLib.cs`): `PascalCase` static methods, `record` result types,
    `SafeHandle` for RAII, `Span<byte>` / `Stream` for binary data, exceptions for errors.

**Why csbindgen (not manual P/Invoke or UniFFI-cs):**

- Generates P/Invoke bindings directly from `extern "C"` functions — leverages existing C FFI
- One-command regeneration when the C API changes
- Production-proven (by Cysharp, used in Unity game development)
- `uniffi-bindgen-cs` (NordSecurity) is less mature and would require a separate UniFFI crate

## Idiomatic C# API

### Result Types (C# Records)

```csharp
public record MetaCodeResult(
    string Iscc, string Name, string MetaHash,
    string? Description, string? Meta);

public record TextCodeResult(string Iscc, int Characters);
public record DataCodeResult(string Iscc);
public record InstanceCodeResult(string Iscc, string DataHash, ulong FileSize);
public record SumCodeResult(string Iscc, string DataHash, ulong FileSize);
// ... one record per gen_*_v0 return type
```

### Public API

```csharp
public static class IsccLib
{
    public static MetaCodeResult GenMetaCodeV0(
        string name, string? description = null,
        string? meta = null, uint bits = 64);

    public static DataCodeResult GenDataCodeV0(ReadOnlySpan<byte> data, uint bits = 64);
    public static DataCodeResult GenDataCodeV0(Stream stream, uint bits = 64);
    public static SumCodeResult GenSumCodeV0(string path, uint bits = 64, bool wide = false);
    public static bool ConformanceSelftest();
    // ... all 32 Tier 1 symbols
}
```

### Streaming Types (IDisposable)

```csharp
public sealed class IsccDataHasher : IDisposable
{
    public IsccDataHasher();
    public void Update(ReadOnlySpan<byte> data);
    public DataCodeResult Finalize(uint bits = 64);
    public void Dispose();
}
```

### Memory Management

- **SafeHandle pattern**: Every native resource wrapped in `SafeHandle` subclass calling the correct
    `iscc_free_*` function. Ensures cleanup even under exceptions and GC finalization.
- **String conversion**: `Marshal.PtrToStringUTF8` → managed `string`, then `iscc_free_string`.
- **Span<byte> for input**: `fixed` statements pin managed arrays — zero-copy to native functions.

### Target DX

```csharp
using Iscc.Lib;

var result = IsccLib.GenMetaCodeV0("Title", description: "A description");
Console.WriteLine(result.Iscc);       // "ISCC:..."
Console.WriteLine(result.Name);       // "Title"

using var hasher = new IsccDataHasher();
hasher.Update(chunk1);
hasher.Update(chunk2);
var code = hasher.Finalize(bits: 64);

var sum = IsccLib.GenSumCodeV0("path/to/file.bin");
```

## Distribution / Publishing (NuGet)

- **Package ID**: `Iscc.Lib`
- **Authentication**: `NUGET_API_KEY` GitHub Actions secret
- **Native libraries**: Bundled as RID-specific content under `runtimes/<rid>/native/`
- **Build pipeline**: Reuses `build-ffi` matrix from `release.yml`, then `dotnet pack` +
    `dotnet nuget push`

## Binding Regeneration

When the C FFI surface changes:

```bash
csbindgen \
    --input ../../crates/iscc-ffi/src/lib.rs \
    --output Iscc.Lib/Native/NativeMethods.g.cs \
    --csharp-dll-name iscc_ffi \
    --csharp-class-name NativeMethods \
    --csharp-namespace Iscc.Lib.Native
```
