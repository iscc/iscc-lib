# Spec: Node.js Bindings ‚Äî Native Addon via napi-rs

An npm package `@iscc/lib` providing native Rust-powered ISCC functions via napi-rs (Rust ‚Üî Node.js
bridge). Follows the same hub-and-spoke model as all other bindings: thin bridge crate wrapping
`iscc-lib` core, with TypeScript declarations for type safety.

## Architecture

**Single-layer design** (simpler than Python/Ruby):

1. **Rust bridge** (`crates/iscc-napi/src/lib.rs`): napi-rs annotated functions that call `iscc-lib`
    core and return JavaScript types (strings, objects, arrays, `Buffer`). All 32 Tier 1 symbols
    exported via `#[napi]` attribute macros.
2. **TypeScript declarations** (`crates/iscc-napi/index.d.ts`): Auto-generated by napi-rs during
    build, providing IDE completion and type checking for consumers.

**Why napi-rs (not wasm-bindgen or N-API C):**

- napi-rs generates native Node.js addons directly from Rust ‚Äî highest performance
- Auto-generates TypeScript declarations from Rust types
- Handles N-API versioning, async, and Buffer types automatically
- Same architecture as many popular npm native packages (e.g., `@napi-rs/canvas`, `swc`)
- Separate from `@iscc/wasm` which targets browser environments

## Rust Bridge Layer

All functions use `#[napi]` attribute macros. Result types are returned as JavaScript objects:

```rust
#[napi(object)]
pub struct MetaCodeResult {
    pub iscc: String,
    pub name: String,
    pub metahash: String,
    pub description: Option<String>,
    pub meta: Option<String>,
}

#[napi]
pub fn gen_meta_code_v0(
    name: String,
    description: Option<String>,
    meta: Option<String>,
    bits: Option<u32>,
) -> napi::Result<MetaCodeResult> { ... }
```

### Naming Convention

napi-rs auto-converts Rust `snake_case` to JavaScript `camelCase`:

- `gen_meta_code_v0` ‚Üí `genMetaCodeV0`
- `text_clean` ‚Üí `textClean`
- `alg_minhash_256` ‚Üí `algMinhash256`

### Streaming Hashers

`DataHasher` and `InstanceHasher` are exposed as JavaScript classes via `#[napi]`:

```rust
#[napi]
pub struct DataHasher {
    inner: Option<iscc_lib::DataHasher>,
}

#[napi]
impl DataHasher {
    #[napi(constructor)]
    pub fn new() -> Self { ... }

    #[napi]
    pub fn update(&mut self, data: Buffer) -> napi::Result<()> { ... }

    #[napi]
    pub fn finalize(&mut self, bits: Option<u32>) -> napi::Result<DataCodeResult> { ... }
}
```

### Error Handling

Errors are mapped to JavaScript `Error` objects via `napi::Error::from_reason()`. The error message
contains the Rust error description.

## Distribution / Publishing

### npm Publishing

- **Scope**: Published under the `@iscc` npm organization
- **Package**: `@iscc/lib`
- **Authentication**: `NPM_TOKEN` GitHub Actions secret

### Platform-Specific Packages

napi-rs uses optional dependencies for platform-specific native addons:

| Platform        | Optional Dependency Package |
| --------------- | --------------------------- |
| linux-x64-gnu   | `@iscc/lib-linux-x64-gnu`   |
| linux-arm64-gnu | `@iscc/lib-linux-arm64-gnu` |
| darwin-x64      | `@iscc/lib-darwin-x64`      |
| darwin-arm64    | `@iscc/lib-darwin-arm64`    |
| win32-x64-msvc  | `@iscc/lib-win32-x64-msvc`  |

The main `@iscc/lib` package declares these as `optionalDependencies`. npm installs only the
matching platform package automatically.
