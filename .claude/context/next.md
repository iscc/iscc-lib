## Implementation Notes

1. **`NapiSumCodeResult` struct** — add a new field:

    ```rust
    pub units: Option<Vec<String>>,
    ```

    The `#[napi(object)]` derive handles `Option<Vec<String>>` as `string[] | undefined` in
    TypeScript. No special serialization needed.

2. **`gen_sum_code_v0` function** — add 4th parameter after `wide`:

    ```rust
    add_units: Option<bool>,
    ```

    Default to `false` with `unwrap_or(false)`. Pass through to `iscc_lib::gen_sum_code_v0`. Map
    `result.units` directly into the `NapiSumCodeResult`.

3. **Tests** — add 3 tests to the existing `gen_sum_code_v0` describe block in `functions.test.mjs`:

    - `addUnits=true` returns `units` as array of 2 ISCC strings
    - Default call (no addUnits) returns `units` as `undefined`
    - Verify `units[0]` starts with `ISCC:` (Data-Code) and `units[1]` starts with `ISCC:`
        (Instance-Code)

4. **Pattern reference** — NAPI uses `Option<bool>` for optional params (JS callers can omit). The
    existing `bits: Option<u32>` and `wide: Option<bool>` params demonstrate this pattern. napi-rs
    maps `Option<Vec<String>>` to `string[] | undefined` automatically.
