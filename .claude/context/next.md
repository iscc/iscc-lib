freshness check so the header never goes stale when the FFI surface changes. This is issue #24 and
the first step in the C FFI DX milestone.

## Implementation Notes

### 1. Generate the header

Run cbindgen to produce the committed header:

```bash
mkdir -p crates/iscc-ffi/include
cbindgen --config crates/iscc-ffi/cbindgen.toml --crate iscc-ffi --output crates/iscc-ffi/include/iscc.h
```

The config in `cbindgen.toml` already specifies language=C, include guard `ISCC_H`, and the correct
sys_includes.

### 2. Update CI workflow

In `.github/workflows/ci.yml`, the `c-ffi` job currently:

```yaml
  - name: Generate C header
    run: cbindgen --crate iscc-ffi -o crates/iscc-ffi/tests/iscc.h
  - name: Compile C test program
    run: >
      gcc -o test_iscc
      crates/iscc-ffi/tests/test_iscc.c
      -I crates/iscc-ffi/tests
      ...
```

Make these changes:

1. **Replace "Generate C header" step with freshness check** — regenerate header to include/ and
    verify it matches the committed version:

    ```yaml
      - name: Check C header freshness
        run: |
          cbindgen --config crates/iscc-ffi/cbindgen.toml --crate iscc-ffi --output crates/iscc-ffi/include/iscc.h
          git diff --exit-code crates/iscc-ffi/include/iscc.h || \
            (echo "::error::iscc.h is stale. Run: cbindgen --config crates/iscc-ffi/cbindgen.toml --crate iscc-ffi --output crates/iscc-ffi/include/iscc.h" && exit 1)
    ```

2. **Update compile include path** — change `-I crates/iscc-ffi/tests` to
    `-I crates/iscc-ffi/include`

### 3. C test still works

`test_iscc.c` already uses `#include "iscc.h"` (no path prefix). The `-I` flag change in gcc is all
that's needed — no source code changes to `test_iscc.c`.
